Arquitetura

Backend em Rust para
máxima performance

O Trading Service foi construído em Rust com Actix-web — garantindo latência mínima, uso eficiente de memória e segurança de tipos em tempo de compilação. Toda comunicação com exchanges é feita via CCXT (Python), unificando APIs distintas em uma interface consistente.

Portfólios e snapshots são persistidos no MongoDB Atlas, possibilitando cálculo de PnL histórico com precisão. O app mobile consome os dados via API REST segura com autenticação JWT.

Segurança

Seus ativos protegidos
em cada camada

Projetado desde o início com segurança como prioridade. Seus dados financeiros nunca trafegam sem criptografia.

🔒
Expo Secure Store
Chaves de API e tokens armazenados com criptografia nativa no Keychain (iOS) e Keystore (Android). Nunca expostos em plain text ou logs de sistema.
🤳
Autenticação Biométrica
Face ID e Touch ID via Expo Local Authentication. Login automático seguro com fallback para senha. Bloqueio automático por inatividade.
🛡️
OAuth 2.0 + JWT
Login com Google e Apple via OAuth 2.0. Tokens JWT validados e assinados no backend Rust antes de qualquer operação sensível.
🔑
Somente leitura por padrão
Integrações com exchanges operam com permissões de leitura. Nenhuma ordem é criada, modificada ou cancelada sem ação explícita e confirmada do usuário.
